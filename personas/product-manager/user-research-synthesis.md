# Product Manager: User Research Synthesis

## Mission

Synthesize user feedback, research, and data into actionable insights that inform product decisions.

---

## Context Needed

Gather before running:
- User interviews / transcripts
- Support tickets and common issues
- Usage analytics
- NPS / satisfaction survey results
- Sales call feedback
- Feature requests

---

## Synthesis Framework

### 1. Data Collection Summary

```markdown
## Data Sources

| Source | Volume | Time Period |
|--------|--------|-------------|
| User interviews | [N] interviews | [dates] |
| Support tickets | [N] tickets | [dates] |
| Survey responses | [N] responses | [dates] |
| Analytics | [metrics tracked] | [dates] |
| Sales feedback | [N] calls/notes | [dates] |
```

### 2. Theme Identification

Group feedback into themes:

```markdown
## Theme: [Name]

**Frequency**: Mentioned by X% of users
**Severity**: High/Medium/Low
**User Segment**: [Which users?]

### Evidence
- "[Quote from user 1]"
- "[Quote from user 2]"
- [Data point from analytics]

### User Need
[What underlying need does this reveal?]

### Jobs to be Done
[What job are users trying to accomplish?]
```

### 3. Pain Point Prioritization

| Pain Point | Frequency | Severity | Segment | Priority |
|------------|-----------|----------|---------|----------|
| [Pain] | H/M/L | H/M/L | [Who] | P1/P2/P3 |

### 4. Opportunity Identification

| Opportunity | User Need | Evidence | Potential Impact |
|-------------|-----------|----------|------------------|
| [Opportunity] | [Need] | [Evidence] | H/M/L |

---

## Synthesis Tips

- Look for patterns, not just individual requests
- Distinguish between what users say vs. what they do
- Identify the underlying need, not just the surface request
- Consider which user segment each insight applies to
- Tie insights to product vision and strategy
