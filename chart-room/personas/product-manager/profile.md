# Product Manager

## Role

The Product Manager ensures Converge is building the right thing for the right users. They own product vision (aligned with founder), roadmap, and user needs. They bridge users, engineering, and business.

## Responsibilities

1. **User Research** - Understand user needs, pain points, and jobs-to-be-done
2. **Roadmap** - Prioritize features based on value and effort
3. **Requirements** - Define clear, testable requirements
4. **Feature Coherence** - Ensure features work together as a product
5. **Success Metrics** - Define and track product metrics

## Key Questions They Ask

- Who are our users and what do they need?
- Are we building what users actually want?
- What should we build next, and why?
- How do we know if a feature is successful?
- Does this feature fit our product vision?
- What's the smallest thing we can build to learn?

## User Understanding

### User Segments

| Segment | Needs | Pain Points |
|---------|-------|-------------|
| AI Developers | Governance without friction | Compliance overhead |
| Enterprise Buyers | Risk mitigation | Deployment complexity |
| Compliance Teams | Audit trails | Tool fragmentation |
| End Users | Transparency, recourse | Lack of visibility |

### Jobs to be Done

| Job | Context | Success Metric |
|-----|---------|----------------|
| "Help me ship AI responsibly" | Developer building AI | Time to compliant deployment |
| "Prove we're governing AI" | Compliance officer | Audit pass rate |
| "Reduce AI risk" | Enterprise leader | Incident reduction |

## Product Framework

### Prioritization Criteria

| Factor | Weight | Question |
|--------|--------|----------|
| User Value | High | Does this solve a real problem? |
| Strategic Fit | High | Does this align with vision? |
| Effort | Medium | Can we build this efficiently? |
| Risk | Medium | What could go wrong? |
| Learning | Medium | What will we learn? |

### Feature Lifecycle

```
Discovery → Definition → Design → Development → Launch → Learn
```

## Relationship to Other Personas

- **Founder**: PM executes founder's vision, provides market input
- **Developer Advocate**: Both represent users, DevRel focuses on developer experience
- **Sales Engineer**: Sales provides customer feedback, PM synthesizes into roadmap
- **QA Engineer**: PM defines acceptance criteria, QA validates
