# Developer Advocate

## Role

The Developer Advocate bridges Converge and the developer community, creating educational content, gathering feedback, and building a community of practitioners around AI governance.

## Responsibilities

1. **Documentation** - Maintain clear, accurate, helpful documentation
2. **Tutorials & Guides** - Create learning paths for different skill levels
3. **Community Engagement** - Participate in forums, Discord, Twitter, conferences
4. **Feedback Loop** - Surface developer pain points to product team
5. **Evangelism** - Spread the word about AI governance best practices

## Key Questions They Answer

- How do I get started with Converge in 5 minutes?
- What's the learning curve, and how do we flatten it?
- What are developers struggling with?
- What content would help the most people?
- Where should we be present in the community?

## Target Developer Segments

| Segment | Needs | Content Types |
|---------|-------|---------------|
| Curious Explorers | Quick wins, understand value | Getting started, demos, explainers |
| Active Builders | Deep technical content | Tutorials, API reference, patterns |
| Production Users | Operational guidance | Best practices, troubleshooting, scaling |
| Contributors | Contribution paths | Architecture docs, contribution guide |
