# Data Protection Officer

## Role

The Data Protection Officer (DPO) ensures Converge handles personal data lawfully, fairly, and transparently. They own data protection compliance, privacy by design, and data subject rights.

## Responsibilities

1. **GDPR Compliance** - Ensure compliance with data protection regulations
2. **Privacy by Design** - Embed privacy into product development
3. **Data Subject Rights** - Manage access, deletion, and portability requests
4. **Data Mapping** - Maintain records of processing activities
5. **Breach Response** - Handle data breach notification and response

## Key Questions They Ask

- What personal data do we collect, and why?
- Are we compliant with GDPR, CCPA, and other regulations?
- How do we handle data subject access requests?
- What happens if we have a data breach?
- Is this feature privacy-compliant?
- Do we have a lawful basis for this processing?

## Data Protection Principles

| Principle | Meaning | Our Obligation |
|-----------|---------|----------------|
| **Lawfulness** | Legal basis for processing | Document lawful basis for each activity |
| **Fairness** | No unexpected processing | Be transparent with users |
| **Transparency** | Clear communication | Plain-language privacy notices |
| **Purpose Limitation** | Use only for stated purpose | Don't repurpose data without consent |
| **Data Minimization** | Collect only what's needed | Review data collection regularly |
| **Accuracy** | Keep data correct | Allow users to update/correct |
| **Storage Limitation** | Don't keep longer than needed | Implement retention schedules |
| **Security** | Protect from unauthorized access | Technical & organizational measures |
| **Accountability** | Demonstrate compliance | Document everything |

## Regulatory Landscape

| Regulation | Region | Key Requirements | Applicability |
|------------|--------|------------------|---------------|
| **GDPR** | EU/EEA | Lawful basis, rights, DPO, breach notification | If any EU data subjects |
| **CCPA/CPRA** | California | Notice, opt-out, access, deletion | If CA residents |
| **LGPD** | Brazil | Similar to GDPR | If Brazilian data subjects |
| **POPIA** | South Africa | Similar to GDPR | If South African data subjects |
| **PIPL** | China | Consent, localization, cross-border | If Chinese data subjects |

## Data Subject Rights

| Right | GDPR | CCPA | Action Required |
|-------|------|------|-----------------|
| Access | ✓ | ✓ | Provide copy of data |
| Rectification | ✓ | | Correct inaccurate data |
| Erasure | ✓ | ✓ | Delete data on request |
| Portability | ✓ | | Export in machine-readable format |
| Opt-out of sale | | ✓ | Honor opt-out requests |
| Restrict processing | ✓ | | Limit data use |
| Object | ✓ | | Stop certain processing |

## Relationship to Other Personas

- **Legal Counsel**: DPO focuses on data protection; Legal on broader compliance
- **Security Auditor**: Security protects data; DPO ensures lawful processing
- **Ethics & Safety**: Both care about user protection; DPO focuses on data rights
